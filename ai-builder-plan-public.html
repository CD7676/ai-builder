Sprint2-003 / Workflow — new Shipped

Email triage runs without me, drafts replies in my voice, and only asks for a star to override.

What's live
Two scheduled Claude Code /schedule remote triggers running on Anthropic infrastructure — no laptop required. Evening Digest (8:10 PM ET, Sun–Thu) reads the day's inbox via the first-party Gmail MCP connector, categorizes every thread against ~60 label rules, drafts voiced replies for personal correspondence, surfaces unsubscribe candidates with a 48h grace window, and emails an HTML digest. Morning Cleanup (10 AM Tue–Fri, 12 PM Mon) auto-unsubscribes candidates that survived the grace window via WebFetch, then labels and archives every unstarred Auto-Digest thread.
How it works
Star-to-keep is the only gesture — anything unstarred gets labeled, archived, or unsubscribed on the next pass. An Auto-Digest marker label scopes cleanup to threads the digest has already touched, which protects the 14k legacy inbox from accidental archiving. Voice rules are duplicated inline from a local email-voice skill because remote /schedule triggers can't invoke local user skills. Drafts only — nothing is sent without review.

Sprint2-003 · What I built

Two scheduled Claude Code agents that triage Gmail without me being there.

Evening Digest — flow

01
/schedule cron fires at 8:10 PM ET (Sun–Thu)Anthropic infrastructure spawns the agent — no laptop required. Friday and Saturday evenings are intentionally skipped.
02
Search threadsGmail MCP search_threads with in:inbox newer_than:2d -label:"Auto-Digest". Idempotent re-runs and a 24h cushion against missed days.
03
Categorize each threadMatch against ~60 routing rules in priority order, first match wins. Apply the Auto-Digest marker only — the category label and archive happen at cleanup, after a chance to star.
04
Draft voiced repliesFor non-Jobs threads where someone else sent the most recent message: draft a reply using register-aware rules from a local email-voice skill. Drafts only — never auto-sent.
05
Identify unsubscribe candidatesNewsletters and marketing-tone Reviewed/Education threads get Unsub-Suggested. 48-hour grace window: candidates must appear unstarred in two consecutive digests before any irreversible action.
06
Build the HTML digestSections for Needs Your Attention, Already Addressed, per-label groups, Unsubscribe Suggestions (split into final-notice and first-suggestion), and Jobs.
07
Create draft, optionally auto-sendFirst-party connector creates the digest draft. If a community send-capable server is configured, it sends; otherwise I send with one click.

Morning Cleanup — flow

01
/schedule cron fires at 10 AM ET (Tue–Fri) or 12 PM ET (Mon)The Monday-noon routine handles the larger Sunday-evening digest covering Fri+Sat+Sun.
02
Process expired unsubscribe candidatesQuery label:"Unsub-Suggested" -is:starred older_than:1d, extract the unsubscribe URL, hit it via WebFetch, archive on success. Failures (login walls, multi-step forms) surface in the summary for manual handling.
03
Find unstarred Auto-Digest threadsQuery in:inbox -is:starred label:"Auto-Digest" -label:"Unsub-Suggested". Protects 14k legacy emails (no marker, no touch) and respects the unsub grace window.
04
Categorize and archiveApply each thread's category label, remove INBOX to archive. The Auto-Digest marker persists as a permanent breadcrumb.

What triggers a draft

A thread gets a voiced reply draft only when (a) it isn't categorized as Jobs (Jobs are always automated alerts), (b) someone else sent the most recent message, and (c) the email reads like personal correspondence worth a reply. Marketing, alerts, and one-way newsletters never get drafted.

Sprint2-003 · Documentation

Tools used.

  • Claude Code /schedule — remote triggers running on Anthropic infrastructure. No local machine required.
  • First-party Anthropic Gmail MCP connector — search_threads, get_thread, create_draft, list_labels, label_thread, unlabel_thread. Auth handled by the connector once enabled in Claude.
  • WebFetch — one-click unsubscribe execution for candidates that survive the 48-hour grace window.
  • email-voice Claude Code skill — canonical voice rules for reply drafts. Duplicated inline in the digest prompt because remote /schedule triggers can't invoke local user skills.
  • Community Gmail MCP server (optional) — taylorwilsdon/google_workspace_mcp adds auto-send. Without it, the digest falls back to draft-only.
